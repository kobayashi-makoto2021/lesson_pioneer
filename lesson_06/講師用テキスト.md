# 講師用テキスト — レッスン06: 見つけた宝を記録しよう — リスト（配列）

## 注意点

- レッスン05のプログラムを改造して進める。事前に生徒のファイルを読み込ませておく。
- toioは2台使用。キューブ（toio1）が探索役、クリスタル（toio2）が宝物役。
- リストは「すべてのスプライト用」で作成させること。「このスプライトのみ」にすると他スプライトから操作できない。
- ステップ5でクリスタルの旗が押されたときのプログラム（旧：乱数で宝物設定）は不要になる。削除させること。

## よくあるつまずきポイント

**ステップ1：座標の文字結合**

`x列 と , と y行` の `と` でつなぐ文字結合は初めて見るパターン。`と言う` ブロックに直接はめ込ませるとわかりやすい。

`と2秒言う` ではなく `と言う` を使う点に注意。`と2秒言う` は2秒後に消えてしまうため、移動中に表示が途切れる。

**ステップ2：リストの隠し方**

リストをステージ上で右クリック→「隠す」で非表示にする。変数の「表示する」チェックボックスとは操作が異なるため、迷う生徒が多い。

**ステップ3：Button1（緑ボタン）**

「マウスが押された ではない まで待つ」パターンはレッスン02のフラグ制御と同じ構造。詰まる生徒にはレッスン02を参照させる。

完成形：`リストを表示する` → `マウスが押された ではない まで待つ` → `リストを隠す`

![ステップ3完成](image/講師テキスト用完成プログラム/Screenshot%20from%202026-05-24%2019-58-46.png)

**ステップ4：Button5（バツボタン）**

ステップ3との違いは「隠す」の代わりに「すべてを削除する → 表示する → 2秒待つ → 隠す」の順になる点。削除後も少し表示して「空になった」ことを確認できる構造になっている。削除前に表示する順番を間違える生徒が多い。

完成形：`リストを表示する` → `マウスが押された ではない まで待つ` → `すべてを削除する` → `リストを表示する` → `2秒待つ` → `リストを隠す`

![ステップ4完成](image/講師テキスト用完成プログラム/Screenshot%20from%202026-05-24%2019-58-59.png)

**ステップ5：クリスタルの修正**

旗が押されたときの旧プログラム（乱数で宝物設定）は削除させる。残しておくと旗を押すたびに宝物座標が上書きされ混乱の原因になる。

toio2を置いてから1秒待ってから座標を読み取る構造にする。置いた直後は座標が安定しないことがある。

完成形：`toio2 マットに触れたとき` → `1秒待つ` → `宝物x を [toio2] マスの列番号 にする` → `宝物y を [toio2] マスの行番号 にする` → `x座標を 40 × 宝物x、y座標を 40 × 宝物y にする` → `音を鳴らす` → `表示する` → `3秒待つ` → `隠す`

![ステップ5完成](image/講師テキスト用完成プログラム/Screenshot%20from%202026-05-24%2019-58-29.png)

**ステップ6：たからチェックの修正**

`すべてを止める` の前にリストへの追加を入れないと記録されずに終わる。追加→ゴール音→停止の順を守らせる。

文字結合 `x列 と , と y行` はステップ1と同じパターン。気づかせると自信につながる。

完成形：`もし x列 = 宝物x かつ y行 = 宝物y なら` → `x列 と , と y行 を たからリストに追加する` → `リストを表示する` → `Goal Cheer の音を鳴らす` → `リストを隠す` → `すべてを止める`

## 補足説明

**リストとフラグ変数の違い**

フラグ変数は「今どういう状態か」を1つの値で管理するが、リストは「これまでに何があったか」の履歴を複数保存できる。このレッスンの発見ログはリストの履歴としての使い方の典型例。

**次回レッスンへのつながり**

レッスン07ではリストを2本使って座標ペアを管理する。「宝物の場所をあらかじめリストに入れておけば、何個でも設定できる」という問いかけで次回への興味をつなぐ。

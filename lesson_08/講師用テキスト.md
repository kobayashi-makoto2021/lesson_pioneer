# 講師用テキスト — レッスン08: キュー — 順番に巡ろう

## 注意点

- レッスン06のリスト操作（追加・削除・長さ）が前提。詰まる生徒にはレッスン06を参照させる。
- キューブ2が「登録役」、キューブ1が「巡回役」という役割分担を最初に板書or口頭で整理してから作業に入る。
- キューはリストを使って実装する。座標ペアを「列のリスト」「行のリスト」の2本で管理する点を最初に示しておくと、ステップ2〜3で迷いにくい。
- enqueue（追加）はリストの末尾、dequeue（取り出し）はリストの1番目。方向を混同しやすいので、概念説明時に図を示しながら確認する。

## よくあるつまずきポイント

**enqueue・dequeueの方向を混同する**

「追加は末尾・取り出しは先頭」を図で繰り返し確認させる。toio Doの `〜に追加する` ブロックはリストの末尾に追加される。取り出しは `1番目` を指定する。

**dequeueを「読む→削除」の2ステップに分けることに気づかない**

「変数に入れてから削除する」順番を間違えると、削除した後に値を読もうとして空になる。

正しい順：`x番地 を 列のリストの1番目 にする` → `列のリストの1番目を削除する`

逆にしてしまう生徒が多いので、ステップ3の実装前に口頭で確認する。

**2本のリストの長さがずれる**

行を追加したら列も追加する、行を削除したら列も削除する、必ずペアで操作することを徹底させる。片方だけ操作すると長さがずれてバグの原因になる。

**Button5（リセット）の処理順を間違える**

レッスン06のButton5と同じパターン（表示→確認メッセージ→削除→表示→待つ→隠す）。レッスン06を参照させると気づきやすい。

## 補足説明

**キューとリストの違い**

リストはどこにでも追加・削除できる汎用構造。キューは「末尾に追加・先頭から削除のみ」というルールを守ったリストの使い方。ルールを守ることで順番が保証される。

**dequeueの実装パターン**

```
x番地 を 列のリストの1番目 にする   ← 読む
y番地 を 行のリストの1番目 にする   ← 読む
番地に向かう                          ← 移動
列のリストの1番目を削除する           ← 捨てる
行のリストの1番目を削除する           ← 捨てる
```

「読んでから捨てる」の順を崩さないこと。

**キューとスタックの比較（次回以降の伏線）**

キューは FIFO（First In First Out）。スタックは LIFO（Last In First Out）で、後から入れたものが先に出る。「押し込む／飛び出す」イメージ。レッスン12前後で登場するので、余裕があれば軽く触れる程度でよい。

**現実世界との接続**

コンビニのレジ待ち・プリンターの印刷待ち・ゲームのイベント処理はすべてキュー。「順番を守る」仕組みとして日常生活と結びつけると定着しやすい。

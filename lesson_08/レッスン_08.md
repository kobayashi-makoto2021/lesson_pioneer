# レッスン08: キュー — 順番に巡ろう

## できるようになること
- [ ] キュー（Queue）が何かを説明できる
- [ ] enqueue（追加）とdequeue（取り出し）の意味がわかる
- [ ] キューを使ってtoioを順番どおりに巡回させることができる

## このレッスンで必要な知識

### キュー（Queue）とは？

![キューとは](image/ChatGPT%20Image%202026年6月17日%2012_11_02.png)

キューは「**後ろから入れて、前から取り出す**」データ構造だよ。コンビニのレジの列と同じで、先に並んだ人が先に処理される。

**データを追加する（Enqueue ／ エンキュー）**

![Enqueue](image/ChatGPT%20Image%202026年6月17日%2012_12_01.png)

データは後尾（Rear）から追加されるんだ。

**データを取り出す（Dequeue ／ デキュー）**

![Dequeue](image/ChatGPT%20Image%202026年6月17日%2012_13_28.png)

データは先頭（Front）から取り出されるよ。取り出したデータはリストから消える。

**FIFO（ファイフォ）**

![FIFO](image/ChatGPT%20Image%202026年6月17日%2012_16_23.png)

最初に入ったものが、最初に出る。これを **FIFO（First In First Out）** というよ。

**キューの動きをまとめると…**

![キューの動きを見てみよう](image/ChatGPT%20Image%202026年6月17日%2012_16_32.png)

## ミッション

キューブ2（登録役）がマットの上で踏んだ場所を、キューブ1（巡回役）が踏んだ順番どおりに訪問するプログラムを作ろう！

### ステップ1：配布ファイルを開いて確認しよう

`08キュー ：順番に巡ろう.sb3` を開いて、スプライト一覧を確認しよう。

以下の5つのスプライトがあることを確かめてね：

| スプライト | 役割 |
|---|---|
| キューブ | キューから順番に取り出して巡回する（巡回役） |
| キューブ2 | マットを踏んでキューに登録する（登録役） |
| Button1 | 押している間だけキューの中身（リスト）を表示する |
| Button5 | キューをリセットする |
| リストの中身を消す | リストの中身をすべて消す |

### ステップ2：「キューに追加」ブロックを完成させよう

キューブ2スプライトを選ぼう。「**キューに追加**」ブロックの定義がまだ空になっているよ。

![キューに追加の定義（空）](image/Screenshot%20from%202026-06-17%2019-45-11.png)

4つのヒントブロックを使って、toio2を置いた位置をキューに追加するプログラムを作ってみよう。

> **考えるヒント：** toio2がいるマスの「行番号」と「列番号」を、それぞれ行のリスト・列のリストに追加すればいいんだ。

### ステップ3：「キューから取り出す」ブロックを確認しよう

キューブ1スプライトを選んで、「**キューから取り出す**」ブロックの中身を見てみよう。

![キューから取り出す](image/Screenshot%20from%202026-06-17%2019-44-35.png)

**読む → 移動する → 消す** の順番で動かすのが、 dequeue（取り出す）の動きだよ。
取り出したあとにリストから消すのがポイント！

完成したら旗を押してみて、キューブ2でマットを何か所か踏んだあと、キューブ1がどう動くか確かめよう。

### ステップ4：アクティビティ1 — 縦一列で順番を確かめよう

**やること：** マットの同じ列を**上から順番に4マス**、キューブ2で踏んで登録しよう。

旗を押す前に予測してみよう。

> 「キューブ1はどの順番で動くと思う？」

予測したら旗を押して確かめよう。

---

今度は**下から上に向かって**同じ4マスを踏んで登録してみよう。

旗を押す前にまた予測。キューブ1の動きはどう変わるかな？

> **まとめ：** 登録した順番 ＝ 動く順番。これが FIFO！

### ステップ5：アクティビティ2 — 自分だけのルートを作ろう

今度は縦一列じゃなくて、マット上の好きな場所を4〜5か所選んでキューブ2で踏んで登録しよう。

キューブ1は登録した順番どおりに巡回するよ。思いどおりのルートで動いたら成功！

Button5を押すとキューがリセットされるので、何度でも試せるよ。

**チャレンジ：** 4か所を巡回するのに、なるべく短い距離で回れるルートを考えてみよう。どの順番で登録すればいいかな？

## まとめ

- **キュー**は「後ろから入れて、前から取り出す」データ構造
- 入れた順番どおりに出てくる ＝ **FIFO（First In First Out）**
- データを追加する操作を **enqueue**、取り出す操作を **dequeue** という
- キューはゲームのイベント処理・プリンターの印刷待ちなど、日常のいろんな場所で使われているよ

# 講師用テキスト — レッスン07: 復習と自由制作の企画 — 好きなものを作ろう！

## 注意点

- このレッスンはプログラミング作業なし。復習＋企画立案に集中する。
- 9つの概念を全員に一斉に読ませるのではなく、各自のペースで読み進めさせながら質問を拾う形が望ましい。
- 企画は「完璧でなくてよい」ことを最初に伝える。次回以降の制作の中で変更してよい。

## よくあるつまずきポイント

**アイデアが全く浮かばない**

「好きなゲームを思い浮かべて、その一部をtoioで再現するとしたら？」と問いかけると動き出しやすい。あるいは「宝探しゲームを改造するとしたら、どう変える？」と既存のプログラムを起点にする。

**概念を選べない（何でも使おうとする）**

「まず一番使いたいもの1つだけ選んで」と絞り込ませる。全部使おうとすると企画が複雑になりすぎて制作フェーズで詰まる。

**「ステートマシン」「全探索」などの言葉を忘れている**

テキストの復習セクションを見返させる。定義を暗記させる必要はなく、「そういう仕組みを使う」という認識があればよい。

## 時間配分の目安

| 内容 | 目安 |
|---|---|
| 復習セクションを読む（9つの概念） | 15分 |
| ステップ1：アイデアを考える | 15分 |
| ステップ2・3：概念を選んで仕組みを考える | 20分 |
| 全体共有・フィードバック | 10分 |

## 補足説明

**企画のフィードバック観点**

- 概念が2つ以上選ばれているか
- 選んだ概念が企画の中で意味ある役割を持っているか（「カウンターを使うけどどこで？」を確認する）
- 60分×3〜4回のレッスンで完成できそうなスコープか

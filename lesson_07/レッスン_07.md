## できるようになること

---

## 復習 — これまでに習った概念

### ループアンドスイッチ
**→ レッスン01**

「ずっと繰り返しながら、条件に応じて動きを切り替える」パターンだよ。

```
ずっと
  もし ボタンが押されている なら
    ...
```

ゲームのキャラクターがボタン入力をずっと待っているのと同じ仕組みで、インタラクティブなプログラムの骨格になる。

---

### カウンター変数
**→ レッスン01**

数を数えるための変数だよ。「1ずつ増やす」を繰り返すことで、ボタンを押した回数や宝を見つけた数を記録できる。

```
カウント を 0 にする
カウント を 1 ずつ変える
```

---

### フラグ変数 / ループアンドフラグ
**→ レッスン02**

「今どの状態か」を覚えておくための変数だよ。値は「0（オフ）」か「1（オン）」の2種類だけ。

フラグを使うと、「ボタンを押すたびにオン・オフを切り替える」「条件が満たされるまでループを続ける」といった制御ができるよ。

```
もし フラグ = 0 なら
  フラグ を 1 にする
でなければ
  フラグ を 0 にする
```

---

### 乱数
**→ レッスン03**

「1から10までのランダムな数」のように、毎回違う数を取り出す仕組みだよ。予測できない動きや、ゲームのランダムイベントに使える。

---

### ランダム探索
**→ レッスン03**

乱数を使ってでたらめに動き回りながら目標を探す方法だよ。運が良ければすぐ見つかるけど、運が悪いと時間がかかる。全探索と比べてどちらが効率的か考えてみよう。

---

### 二重ループ
**→ レッスン04**

ループの中にループを入れる構造だよ。「x列を−4から4まで動かしながら、それぞれのxでy行を−4から4まで動かす」ように、2方向に繰り返したいときに使う。

```
-4 から 4 まで繰り返す（x列）
  -4 から 4 まで繰り返す（y行）
    ...
```

---

### ステートマシン
**→ レッスン05**

プログラムの「場面」を変数で管理するパターンだよ。「移動中 = 1」「チェック中 = 2」のように状態に番号をつけて、ステップが切り替わるたびに変数を変える。

複数の処理を順番に進めたいときに使えるよ。

---

### 全探索
**→ レッスン05**

マットのマス目を全部しらみつぶしに調べて、目標を必ず見つける方法だよ。二重ループとステートマシンを組み合わせて作る。

「絶対に見つかる」のが強みだけど、マスの数だけ時間がかかる。

---

### 1次元リスト
**→ レッスン06**

複数のデータをまとめて保存できる箱だよ。変数は1つしか覚えられないけど、リストは何個でも追加できる。

```
たからリスト に "2,3" を追加する
たからリスト の 1 番目
たからリスト の長さ
```

## ミッション — 作るものを企画しよう！

上の9つの概念の中から**2つ以上**を選んで、toio＋toio Doで動くプログラムを企画しよう。

### ステップ1: アイデアを考えよう

どんなプログラムを作りたいか、下の欄に書いてみよう。

```
タイトル：

どんなプログラム？（一言で）：

何が面白い？：
```

まだアイデアが浮かばない人は、こんな切り口で考えてみよう。

- toioをどう動かしたい？（探索・ランダム・カウント…）
- 勝ち負けはある？ゴールはある？
- 1人で遊ぶ？2人で遊ぶ？

---

### ステップ2: 使う概念を選ぼう

企画に使う概念にチェックを入れよう（2つ以上）。

- [ ] ループアンドスイッチ（レッスン01）
- [ ] カウンター変数（レッスン01）
- [ ] フラグ変数 / ループアンドフラグ（レッスン02）
- [ ] 乱数（レッスン03）
- [ ] ランダム探索（レッスン03）
- [ ] 二重ループ（レッスン04）
- [ ] ステートマシン（レッスン05）
- [ ] 全探索（レッスン05）
- [ ] 1次元リスト（レッスン06）

---

### ステップ3: 仕組みを考えよう

選んだ概念がプログラムの中でどんな役割を持つか、書いてみよう。

```
概念①「          」→ この場面で使う：

概念②「          」→ この場面で使う：

概念③「          」→ この場面で使う：（あれば）
```

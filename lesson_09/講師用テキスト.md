# 講師用テキスト — レッスン09: スタック — 上から順に取り出す

## 注意点

- レッスン08（キュー・FIFO）を前提とする。キューの enqueue/dequeue の実装が定着していない生徒は本レッスンで混乱しやすい。
- 登録役（キューブ2）の push ブロックはレッスン08の enqueue と全く同じ実装。違いは pop 側だけであることを冒頭に伝えると混乱が減る。
- 本レッスンの核心は「登録した順番と**逆**に動く」という体験。アクティビティ前に必ず予測させてから実行させること。

## 概念説明で使うスライド

授業冒頭で順番に見せる。

### スタックとは？

![スタックとは？](image/ChatGPT%20Image%202026年6月17日%2013_58_06.png)

「上に積んで、上から取り出す」データ構造。一番上のデータを **Top（トップ）** と呼ぶことを板書する。

### Push（データを追加する）

![Push](image/ChatGPT%20Image%202026年6月17日%2013_59_09.png)

Top に積み上げる操作。「いつも Top に追加」を繰り返す。

### Pop（データを取り出す）

![Pop](image/ChatGPT%20Image%202026年6月17日%2013_59_21.png)

Top から取り出す操作。取り出したデータはスタックから消える点を強調する。

### LIFO

![LIFO](image/ChatGPT%20Image%202026年6月17日%2013_59_27.png)

Last In First Out。「最後に入れたものが最初に出る」。キューの FIFO と対比させる。

![スタックの動きを見てみよう](image/ChatGPT%20Image%202026年6月17日%2013_59_34.png)

## よくあるつまずきポイント

**FIFO と LIFO を混同する**

「どこから取り出すか」だけが違う。enqueue と push は同じ（末尾に追加）、dequeue は先頭・pop は末尾。キューとスタックの比較スライドを見せながら説明する。

![キューとスタックを比べよう](image/ChatGPT%20Image%202026年6月17日%2014_00_06.png)

**pop の実装で「読む→削除」の順を間違える**

レッスン08と同じ落とし穴。「変数に入れてから消す」順を崩すと、削除後に空の値を読もうとしてバグになる。

正しい順：
```
x番地 を 列のリストの 長さ 番目 にする   ← 読む
y番地 を 行のリストの 長さ 番目 にする   ← 読む
番地に向かう                              ← 移動
列のリストの 長さ 番目を削除する          ← 捨てる
行のリストの 長さ 番目を削除する          ← 捨てる
```

**dequeue（1番目）と pop（長さ番目）を取り違える**

レッスン08の dequeue は `1番目`、pop は `長さ番目`（末尾）。ここの違いが FIFO/LIFO の正体。

**2本のリストの長さがずれる**

行を追加したら列も追加、行を削除したら列も削除。ペア操作の徹底はレッスン08と同じ。

## 補足説明

**push と pop の実装まとめ**

| 操作 | Scratch のブロック | 対象位置 |
|---|---|---|
| push（追加） | `なにかを リストに追加する` | 末尾（長さ+1） |
| pop（取り出し） | `リストの 長さ 番目` を読む → 削除 | 末尾（長さ番目） |

**日常生活のスタック例（概念導入時に使う）**

- 積み上げた本：上から順に取らないと下の本は取れない
- 食器のお皿：後から積んだお皿が上に来て先に取り出される
- ブラウザの「戻る」ボタン：最後に訪れたページから順に戻る

**「なぜ逆順になるの？」への答え方**

A→B→C の順に push するとリストは `[A, B, C]` になる。pop は末尾から取るので C→B→A の順に取り出される。「積み上げた順番の逆が取り出し順」。アクティビティ後にこの原理を改めて確認させると定着しやすい。

**現実世界との接続（まとめ時に使う）**

![スタックはどこで使われる？](image/ChatGPT%20Image%202026年6月17日%2013_59_43.png)

![スタックを使うメリット](image/ChatGPT%20Image%202026年6月17日%2013_59_53.png)

関数の呼び出し（call stack）・ブラウザの戻る機能・テキストエディタの Ctrl+Z はすべてスタック。「最後にやったことを一番先に取り消す」という自然な動作がスタックで実現されていることを伝えると印象に残りやすい。
